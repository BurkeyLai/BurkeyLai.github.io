My First DApp

February 27, 2022 in Blockchain, DApp

In order to understand the blockchain and its application DApp, I finished my first smart contract with Solidity and interacted with it through React based frontend website.

I’m totally a newbie to JavaScript and website developments. In order to fast foward my steps about my learning and simplify what I have experimented, I use frontend lite module to design my static website. Any suggestion and idea would be very welcomed and appreciated.

GitHub

Contract ABI

ABI (Application Binary Interface) is the standard way to interact with contracts in the Ethereum ecosystem, both from outside the blockchain and for contract-to-contract interaction. It allows us to use its interface generated from the smart contract to create a website for interacting with users.

In the previous post , we have our first smart contract. We can fetch the ABI at the bottom of the Solidity compiler tab. Copy it and paste in a file named abi.json, which will be used later.

React Project

I have surveyed an article that teach people create their own NFT and use a website to contact the smart contract to mint. The example in the article use React to build the website and it is very easy to get started.

First, we create the React Project:

npx create-react-app kingofether
cd kingofether
yarn start

Then, it will open a new webpage and you can see a React logo.

Then, we use rewired to avoid some webpack 5 issues.

yarn add react-app-rewired

The package.json should be modified:

"scripts": {
    "start": "react-app-rewired start",
    "build": "react-app-rewired build",
    "test": "react-app-rewired test",
    "eject": "react-scripts eject"
},

Add a file config-overrides.js at the project root and copy & paste the code below:

const webpack = require('webpack');

module.exports = function override(config) {
    const fallback = config.resolve.fallback || {};
    Object.assign(fallback, {
        "crypto": require.resolve("crypto-browserify"),
        "stream": require.resolve("stream-browserify"),
        "assert": require.resolve("assert"),
        "http": require.resolve("stream-http"),
        "https": require.resolve("https-browserify"),
        "os": require.resolve("os-browserify"),
        "url": require.resolve("url")
    })
    config.resolve.fallback = fallback;
    config.plugins = (config.plugins || []).concat([
        new webpack.ProvidePlugin({
            process: 'process/browser',
            Buffer: ['buffer', 'Buffer']
        })
    ])
    return config;
}

Run yarn start again and if there are some errors about missing packages, use yarn add [package name] or npm install [package name] to fix it.

web3.js

As the website said, web3.js is a collection of libraries that allow you to interact with a local or remote ethereum node using HTTP, IPC or WebSocket. By using web3.js, we can manipulate the contract address, the accounts associated with the contract and the ether balance. It allows us to develop kinds of applications based on our smart contract.

yarn add web3

In the index.js:

// import the module
const Web3 = require('web3');

// Socket provider
const web3 = new Web3(new Web3.providers.WebsocketProvider('ws://[RPC Server address provided by web3 provider]'));

Since we still testing using Ganache’s local blockchain, the address can be filled with the provided RPC Server address.

Some other usage about web3.js like web3.eth or web3.eth.Contract can be surveyed in the official document.

Run DApp

Beside web3.js, we use other lite modules for development. Couple of them includes nanohtml, morphdom and csjs-inject. The referred usages can be consider here .

yarn add nanohtml morphdom csjs-inject

Load those module in index.js:

const html = require('nanohtml');
const csjs = require('csjs-inject');
const morphdom = require('morphdom');

The referred source code is provided or a modification version from myself is here . There are something else to do, load abi.json that we have mentioned earlier and define the deployed smart contract address.

const ABI = require('./abi.json');
const DEFAULT_ADDRESS = '[deployed smart contract address]';

For example, we can use Remix’s interface and select one account provided by Ganache to deploy a smart contract. Then copy that smart contract address and paste it here, the variable DEFAULT_ADDRESS

Conclusion

In this post, we have done several things.

  • Generate the abi.json from the smart contract we finished in previous post for later usage.
  • Create the first React project and setting up the environment for launching website.
  • Brief introduction about web3.js and use it in our DApp for development in order to be more convenient.
  • Use JavaScript lite modules to create the user interface and behavior of applications for the DApp website.

We just finished our first DApp and there are still many things else to learn.
