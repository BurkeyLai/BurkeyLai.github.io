My First Smart Contract

February 25, 2022 in Blockchain, Smart Contract

In order to understand the blockchain and its application DApp, I try to experiment my first smart contract with Solidity on Ethereum IDE Remix . To interact with it, a simple website interface created using React and frontend lite module is implemented. The whole experimental environment is built on Ganache , the personal Ethereum chain.

GitHub

Ganache

First of all, the experimental environment need to be setup. Here we use Ganache to test our smart contract function as it was deployed on the blockchain. As Ganache installed, use Quick Start can create 10 testing accounts which are existed only in this local private chain. Each account has its own address, private key and balance for user’s experimental purpose. The detail can be studied at the Ganache’s official website.

The most importance thing here is to remember the RPC Server address diplayed on the top of the user interface. Something like:

RPC SERVER
HTTP://127.0.0.1:XXXX

King of the Ether

King of the Ether was a famous DApp. This game has a simple logic and could be simplifed easily, so I use it for my practice. There are few mechanisms to build this game:

  • A lot of users can join this game, the one who pay the most ether can be the king.
  • Before the termination of the game, any one could be the usurper if he pay more ether with a specific amount than the current king.
  • The usurped king can take the ether paid by usurper, with some management fee deducted.
  • The owner can withdrawal the fee as the game terminated.

I referred the implementation here and its prototype on GitHub . You can just use this example to go forward or try to do some modifications.

I have my own modified version written in Solidity which are just added with some more functions and it’s welcome to provide any suggestion.

Experiment on Remix

Now it’s time to open Remix. The fundamental usage for this IDE can be found on the official website and it won’t be further discussed here.

In the Remix workspace, create your first .sol file and you can just copy and paste the code from my project. The most important thing here is to decide which version of Solidity you are going to use and enable ABIEncoderV2 .

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;
pragma experimental ABIEncoderV2;

The difference of the Solidity version would affect the implementation of syntax so it is needed to be consider carefully.

Next, press the Solidity compiler tab on the sidebar at left hand side. Check your compiler version and compiled language, then press Compile XXX.sol. If there is any error, fix it or the smart contract won’t be interacted.

Then, press Deploy & run trancations tab to go forward. Here, we can launch Ganache first and copy the RPC Server address. At the ENVIRONMENT column which default is JavaScript VM, choose another one, the Web3 Provider. We paste the copied address at the column below and confirm. Now, we can interact with our smart contract with the listed accounts provided by Ganache. For example, use an account as the owner to deploy the smart contract, and switch to other accounts for playing the game.

One more thing to notice about is to enter the amount of days the smart contract exist and the gas you need to pay. The Ethereum Unit Converter can make you more convenient to do the experiment.

Conclusion

In this post, we create our first smart contract written in Solidity with Remix. And do the experiment in the local Ethereun blockchain created using Ganache. Next , we will implement the smart contract through a website to complete a simple DApp.
