First Golang App

March 12, 2022 in Golang, App

One of the popular reason for Golang is about its lightweight and portability, which is convenient to deploy the application on kinds of platforms. In this post, we bring up our first app written in Go and running on Android platform.

Note that the app written in Go can also run on ios which is dependent on your requirement.

Requirements and Process

First of all, writing Go app needs the Golang develop environment. The Go version have to be greater than 1.5 or the gomobile cannot build and run the Go app. My OS is Ubuntu 16.04 LTS and I use gvm to maintain my Go environment which is very clean and convenient. The installation process:

sudo apt-get install binutils bison gcc make build-essential -y
bash < <(curl -s -S -L https://raw.githubusercontent.com/moovweb/gvm/master/binscripts/gvm-installer)
source ~/.gvm/scripts/gvm
gvm install go1.17.5 --binary
gvm use go1.17.5 --default

Moreover, Android Studio also needs to be installed. We require the NDK for our gomobile and it will be discussed later.

The whole process could be:

  1. Write Go app
  2. Build up our go project using gomobile
  3. Use generated .aar and .jar file to build APK
  4. Install the APK into our Android device

So, that’s begin.

Andoird NDK

Install Android NDK. Go to Android Studioâ€™s Tools -> SDK Manager. Under SDK Tools, select the NDK version you want to download and click Apply. Need to ensure that the installed version is supported by gomobile. Here I use the version 23.1.7779620.

Check the Android SDK Location up there and copy it. Modify the ~/.bashrc:

export ANDROID_HOME=Your/Android/SDK/Location
export ANDROID_NDK_HOME=Your/Android/SDK/Location/ndk/23.1.7779620

The NDK environment setting is finished. Remember source ~/.bashrc to update the modifications.

Gomobile

First, we set up gomobile for later use. It is a tool for building and running mobile apps written in Go.

go install golang.org/x/mobile/cmd/gomobile@latest

Update the ~/.bashrc, you can use go env to check your GOPATH.

export PATH=$PATH:$GOPATH/bin

Here we can first use the example app for testing.

mkdir test1
cd test1
go mod init test1
gomobile init
go get golang.org/x/mobile/example/basic
gomobile build -target=android golang.org/x/mobile/example/basic

We can see an APK basic.apk is generated. Now, launch the enumerator and we gonna install the APK.

gomobile install golang.org/x/mobile/example/basic

Click an app named basic and you can see something like this:

Congrats! The tool gomobile works and you have now success to run the example app.

Note that when I use gomobile version to check my version, it failed and I cannot figure what’s going on. However, the example is successful and we can just ignore that error temporary.

Run your own app

Now we are going to build our own app.

mkdir test2
cd test2
go mod init test2
gomobile init
go get golang.org/x/mobile/cmd/gobind
vim test2.go

Copy and paste the code below into test2.go.

package mymodulelib

import "fmt"

func SayHello() {
        fmt.Println("Hello from mymodule")
}

And use the command below to generate .aar and .jar file.

gomobile bind -target android

Now, we create a new android project within Java and go to New -> New Module. Select Import .JAR/.AAR Package then choose .aar file and click Finish. Check the settings.gradle have include the library.

include ':mymodulelib'

Go to build.gradle (:app) and add the line below inside the dependencies.

implementation project(path: 'mymodulelib')

Something notable is that your compileSdkVersion need to be higher enough or it will occur errors when you build APK.

compileSdkVersion 31

In the MainActivity.java, add:

mymodulelib.Mymodulelib.sayHello();

Then, go to Build -> Build Bundle(s) / APK(s) and click Build APK(s).

An APK is now generated, I will copy it to the folder test2 and rename it to test2.apk.

Now we have to install the APK. Use command:

adb install test2.apk

You can now see an app test2 in the enumerator, launch it and open the Logcat in Android Studio. You will see:

Excellent! Now we have our own Golang app running on Android.

Conclusion

In this post, we have go through the process to build our first Golang app executable on Android.

  • Check the requirements and setup the environment.
  • Install Android NDK.
  • Install the tool gomobile and experiments the example app.
  • Writing our own Golang app and deploy it on Android platform.

The further application needs to be well developed, and I will keep on update my trial.
